Usage Guidelines
  • Brand colours (primary, product-purple, product-green, product-red) are for product identity — use for primary buttons, headers, links, and brand accents.
  • Semantic colours (info, danger, success, warning) are for feedback and status. info is intended for informational messages, tips, and neutral help text. Do not use semantic colours for branding.
  • Neutral grays are for text, borders, backgrounds, and disabled states. They provide hierarchy without competing with brand or semantic colours.
  • Each palette includes shades 50–950 to support light/dark modes, hover states, and accessible contrast ratios (WCAG AA minimum).
  • The 600 shade is the canonical "brand" value for each product palette and is used as the default solid button background.

Usage Guidelines
  • Text only: Use when the action is clear from the label alone, and space is limited.
  • Icon only: Use when the action is universally recognisable (e.g., delete, search, save). Ensure tooltip or aria-label.
  • Icon on left: Icon reinforces meaning, read first (LTR). Common for primary actions.
  • Icon on right: Indicates direction or state change after label (e.g., "Next →", "Submit ✓").
  • Icons on both sides: Combines a leading icon for category and a trailing icon for the action result.
  • Split button: Primary action with secondary related action.

Usage Guidelines
  • Basic Card: Use for simple content grouping, feature descriptions, or content blocks.
  • Card with Image: Use for visual content like services, news, or profiles where imagery adds context.
  • Header & Footer: Use for structured data display, forms, or actions that need clear separation.
  • Horizontal Card: Use for detailed listings where content needs more horizontal space.
  • Interactive Card: Use for navigation, feature selection, or dashboard modules with hover feedback.
  • Selectable Card: Use for plan selection, option picking, or multi-choice interfaces.
  • Status Cards: Use for status dashboards, notification summaries, or state indicators.
  • Metric Cards: Use for KPIs, statistics, and dashboard widgets. Keep to 4 per row maximum.
  • Compact Cards: Use for dense data grids, mobile dashboards, or when space is limited.
  • Progress Cards: Use for task tracking, application status, or multi-step process monitoring.
  • Cards should have consistent padding (16px–20px) and use shadow-elevation-1 to shadow-elevation-3 for depth.
  • Interactive cards must have visible focus states and hover feedback for accessibility.

When to use Stepper vs Timeline
  • Stepper: Use when the user is completing a multi‑step process (e.g., wizard, form submission). Shows progress and remaining steps.
  • Timeline: Use to display a chronological log of past events or status history. Shows what happened and when.
